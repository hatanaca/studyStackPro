+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| FILOSOFIA DE TESTES DESTE PROJETO                                                                                                                                                                                                                                                     |
+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
| Testes de feature sobre testes de unidade: Para um projeto CRUD com API REST, Feature Tests que exercitam o endpoint completo (HTTP → banco → resposta) detectam mais bugs reais que mocks excessivos. Unit Tests ficam para lógica complexa isolada: Aggregators, Jobs, Composables. |
|                                                                                                                                                                                                                                                                                       |
| Banco real, não SQLite: PostgreSQL em container de teste. SQLite não suporta schemas (analytics.*), triggers, ou funções específicas do Postgres. Testar com SQLite seria testar algo diferente do que roda em produção.                                                              |
|                                                                                                                                                                                                                                                                                       |
| Event::fake() e Queue::fake() para isolar efeitos colaterais: Feature tests de sessões verificam que o evento foi disparado e o job foi enfileirado — sem executar o job em si. Job tem seu próprio teste unitário.                                                                   |
|                                                                                                                                                                                                                                                                                       |
| RefreshDatabase vs DatabaseTransactions: Usar RefreshDatabase (truncate) por padrão. DatabaseTransactions é mais rápido mas quebra com múltiplas conexões (PostgreSQL schemas separados, Redis). Consistência > velocidade nos testes.                                                |
+---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

2.2 TestCase Base Personalizado

Uma classe TestCase base centraliza configurações comuns: criação de usuário autenticado, helpers de assertion, limpeza de cache.

+-------------------------------------------------------------------------------+
| <?php // tests/TestCase.php                                                   |
|                                                                               |
| namespace Tests;                                                              |
|                                                                               |
| use App\Models\User;                                                          |
|                                                                               |
| use Illuminate\Foundation\Testing\TestCase as BaseTestCase;                   |
|                                                                               |
| use Illuminate\Foundation\Testing\RefreshDatabase;                            |
|                                                                               |
| use Illuminate\Support\Facades\Cache;                                         |
|                                                                               |
| abstract class TestCase extends BaseTestCase                                  |
|                                                                               |
| {                                                                             |
|                                                                               |
| use RefreshDatabase;                                                          |
|                                                                               |
| // Helper: cria usuário e retorna com token de autenticação                   |
|                                                                               |
| protected function createAuthenticatedUser(array $overrides = []): array      |
|                                                                               |
| {                                                                             |
|                                                                               |
| $user = User::factory()->create($overrides);                                  |
|                                                                               |
| $token = $user->createToken('test-token')->plainTextToken;                    |
|                                                                               |
| return compact('user', 'token');                                              |
|                                                                               |
| }                                                                             |
|                                                                               |
| // Helper: faz request autenticado                                            |
|                                                                               |
| protected function asUser(User $user): static                                 |
|                                                                               |
| {                                                                             |
|                                                                               |
| return $this->actingAs($user);                                                |
|                                                                               |
| }                                                                             |
|                                                                               |
| // Helper: request com Bearer token (testa via header, não via actingAs)      |
|                                                                               |
| protected function withBearerToken(string $token): static                     |
|                                                                               |
| {                                                                             |
|                                                                               |
| return $this->withHeader('Authorization', "Bearer {$token}");                 |
|                                                                               |
| }                                                                             |
|                                                                               |
| // Helper: limpa Redis entre testes que testam cache                          |
|                                                                               |
| protected function flushTestCache(): void                                     |
|                                                                               |
| {                                                                             |
|                                                                               |
| Cache::flush();                                                               |
|                                                                               |
| }                                                                             |
|                                                                               |
| // Helper: assertion customizada para envelope de resposta                    |
|                                                                               |
| protected function assertApiSuccess($response, int $status = 200): void       |
|                                                                               |
| {                                                                             |
|                                                                               |
| $response->assertStatus($status);                                             |
|                                                                               |
| $response->assertJsonPath('success', true);                                   |
|                                                                               |
| }                                                                             |
|                                                                               |
| protected function assertApiError($response, int $status, string $code): void |
|                                                                               |
| {                                                                             |
|                                                                               |
| $response->assertStatus($status);                                             |
|                                                                               |
| $response->assertJsonPath('success', false);                                  |
|                                                                               |
| $response->assertJsonPath('error.code', $code);                               |
|                                                                               |
| }                                                                             |
|                                                                               |
| }                                                                             |

8. Checklist — O que Testar em Cada Módulo

  -----------------------------------------------------------------------------------------------------------------------
  Módulo                     Deve ter teste                                                      Tipo        Prioridade
  -------------------------- ------------------------------------------------------------------- ----------- ------------
  POST /api/v1/auth/login           Credenciais corretas, erradas, resposta genérica (sem info leak)    Feature     Alta

  POST /api/v1/auth/logout          Token revogado funciona 401 após logout                             Feature     Alta

  GET /api/v1/technologies          Usuário vê apenas suas tecnologias                                  Feature     Alta

  POST /api/v1/study-sessions       Evento disparado, job enfileirado, duration_min calculado           Feature     Alta

  POST /api/v1/study-sessions/start       Máx 1 sessão ativa (409 na segunda)                                 Feature     Alta

  GET /api/v1/analytics/dashboard   Estrutura correta, cache HIT sem queries ao banco                   Feature     Alta

  UserMetricsAggregator      Streak 0, 1, N, gap, personal best, múltiplas sessões/dia           Unit        Alta

  RecalculateMetricsJob      3 aggregators chamados, evento disparado, failed() loga             Unit        Alta

  analytics.store            fetchDashboard, isFresh, updateFromWebSocket, computed              Unit        Alta

  useSessionTimer            Init com elapsed do server, incremento, HH:MM:SS, unmount cleanup   Unit        Alta

  Autorização cross-user     Usuário B não acessa recursos do usuário A (403)                    Feature     Alta

  Rate limiting              POST /api/v1/auth/login: exceder 3 req/min (por IP) retorna 429       Feature     Média

  Cache invalidation         Criar sessão → cache analytics removido                             Feature     Média

  TechDistributionWidget     Top 6 + Outros, click navega com query param                        Unit/Comp   Média
